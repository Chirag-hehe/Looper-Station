Support

Setup notes.

The browser version is useful for testing the workflow. For stage-grade performance, test with the exact phone, browser, audio interface, and controller you plan to use.

Mic access

Open the site from localhost or HTTPS, then click Enable mic in the control panel. If the browser blocks the prompt, check site permissions in the address bar and reload.

Audio interface

Connect the interface before enabling the mic. After permission is granted, choose the device from the Input menu. Wired headphones reduce feedback and latency significantly on stage.

MIDI controller

Enable MIDI from the control panel. Notes 36–41 or CC 20–25 control tracks 1–6. Note 42 or CC 102 starts playback. Note 43 or CC 103 stops playback. Works in Chrome and Edge; Firefox requires a MIDI browser extension.

Keyboard shortcuts

Space toggles play/stop. Keys 1–6 arm the corresponding track. M toggles the metronome, O toggles overdub, and Q cycles the quantize setting (Bar → Beat → Off).

Saved takes

Saved takes live in the current browser profile using IndexedDB. Clearing site data in your browser settings will delete them. Export WAV files for anything you want to keep outside the browser permanently.

Browser support

Chrome and Edge have the best support for Web Audio, MediaRecorder, and Web MIDI. Firefox supports recording and playback. Safari supports recording on iOS 15+ but does not support Web MIDI. For live performance, Chrome on desktop is recommended.
